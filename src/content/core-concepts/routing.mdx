# Routing

The Epic Stack uses file-based routing with Remix, but with a twist. Instead of using the built-in routing convention of Remix, it employs [remix-flat-routes](https://github.com/kiliman/remix-flat-routes), a special implementation that adds some features, particularly "hybrid routes".

## Configuration

The routing is configured in the `vite.config.js` file at the root of the app.

## Understanding the Route Structure

To understand the current route structure of your application, you can run:

<CodeBlock code={`npx remix routes`} lang='bash' />

This command outputs the routes of your application in a JSX-like format, revealing the routes that will be generated based on your current file structure.

## Benefits of remix-flat-routes

The remix-flat-routes hybrid routing system provides the best of both worlds:

1. Colocation of routes to the code they use
2. Organized folder structure to keep routes together as needed

If you're familiar with the Remix routing convention, you can think of remix-flat-routes as converting `+/` to `.`.

This approach allows for a more flexible and organized route structure while maintaining the benefits of file-based routing.
